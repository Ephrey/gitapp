  • step 1 : git init (initialize git in your local repository folder)
  • step 2 : git config --global user.name 'yourName' (create your user name for all repository)
  • step 3 : git config --global user.email 'yourEmail' (create your email for all reponsitory)
  • step 4 : git add file1 (filee2, file3,... in case you have more than one file) (add file to the staging area, or start track file)
  • step 5 : git add status (shows all tracked (green color) and untracked (red color) file/s)
  • step 6 : git rm --cached fileName (remove a file from the staging area or unstage a file)
  • step 7 : git rm --cached -f fileName (force to remove file in the staging area)
  • step 8 : git add *.fileExtension (add all file with the specified extension (e.g : html, css, js, ...) to the staging area)
  • step 9 : git add . (add all files in the current repository to the staging area)
  • step 10 : git commit (commit all file in the staging area for change tracking). Executing this command line will bring you to a second screen where you insert your comments (type I on the keyboard to start writing). when done click ESC on the keyboard and type :WQ to quit. Now you can see the status of your repository.
  • step 11 : use 'git status' to check the status of your fikes (changed and unchanged)
  • step 12 : git commit -m 'your_comment_goes_here' (the shortcut of the 10th step above)
  • step 13 : to ignore certain files being committed just create a file named .gitignore and add all your files inside.
  • step 14 : like step 8 use *.fileExtension in the .gitignore file to ignore all files with the specify extension
